Archive — comportement impression vouchers avant unification « mode Small » Mikhmon
==================================================================================

Date (approx.) : alignement web + APK sur buildSmallModePrintHtml (print.php small=yes).

Ancien comportement
-------------------
1) printTickets() utilisait buildHtml() : grille desktop 4×8 tickets par « page » +
   zoom @media print sur body, sans layout 48 % / ligne du mode Small.

2) Sur navigateur mobile (hors WebView), Vouchers.tsx et GenerateVouchers.tsx
   appelaient buildTicketPrintHtml(..., mobile: true) → buildHtml avec branche
   « mobile » (colonnes calculées depuis la largeur du ticket), pas le CSS Small.

3) Seul le desktop (fenêtre pré-ouverte) utilisait déjà buildSmallModePrintHtml.

Mise à jour (alignement Mikhmon v3)
-----------------------------------
- buildSmallModePrintHtml utilise désormais le bloc CSS littéral de print.php
  (plus de table.voucher 48 % ni règles « maison »).
- Zoom html uniquement si échelle tenant ≠ 100 %.
- Preset fichier mikhmon.php = gabarit template-small (135 px), comme nanotech-small.php.

Restauration rapide (ancien comportement pré-unification Small)
-----------------------------------------------------------------
- Dans artifacts/app/src/lib/print.ts, rétablir printTickets() pour appeler
  buildHtml(htmlItems, title, false, scale, false) comme avant ce dossier.

- Dans Vouchers.tsx et GenerateVouchers.tsx, branche isMobileBrowser :
  rétablir buildTicketPrintHtml(data.html, title, capturedMobileScale, true).

Réglages serveur / UI (inchangés par cette unification)
--------------------------------------------------------
Les échelles « Small » et « Mobile » (TicketTemplate, API tenant) restent ;
seul le pipeline HTML des vouchers passe partout par le même document Small.
